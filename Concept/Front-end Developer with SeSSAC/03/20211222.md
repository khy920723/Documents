- u태그와 i태그

u태그와 i태그는 잘 쓰지는 않으므로 span태그나 strong을 이용하여 CSS에서 변경해준다

```html
<h1><u>SEOUL BEAUTY</u></h1>
```

> HTML `<u>` 요소는 과거 HTML에서 밑줄(Underline) 요소로 불렸으며 지금도 밑줄을 추가할 목적으로 남용하는 경우가 있습니다. 그러나 밑줄을 위해서는 CSS [`text-decoration`](https://developer.mozilla.org/ko/docs/Web/CSS/text-decoration) 속성을 `underline`으로 지정하는 방법을 대신 사용해야 합니다.

```html
<h2><i>face of k-beauty</i> | 서울의 아름다움, 디지털을 타고 전 세계로 퍼지다.</h2>
```

> **HTML `<i>` 요소**는 텍스트에서 어떤 이유로 주위와 구분해야 하는 부분을 나타냅니다. 기술 용어, 외국어 구절, 등장인물의 생각 등을 예시로 들 수 있습니다. 보통 기울임꼴로 표시합니다.

- 요소 레벨 변경

CSS [`display`](https://developer.mozilla.org/ko/docs/Web/CSS/display) 속성을 사용해 요소의 시각적 표현 레벨을 바꿀 수 있습니다. 예컨대 `display`의 값을 `inline`에서 `block`으로 바꾸면, 브라우저에게 이 인라인 요소를 인라인 박스 대신 블록 박스를 사용해 그리라고 알리는 것입니다. 그러나 이 방법을 사용해도 요소의 카테고리와 콘텐츠 모델은 바뀌지 않습니다. 즉 `<span>` 요소의 `display`를 `block`으로 지정한다 해도, 그 안에 `<div>`를 넣을 수는 없습니다.

> 이미지는 inline 요소이므로 정확히 가운데에 위치시키고 싶다면 div로 묶은 다음 가운데 정렬 가능

```html
<div>
    <img src="img/beauty/beauty1.jpg" alt="">
</div>
```

- baseline (베이스라인)
  1. baseline : 기준선
  2. descender : 기준선에서 내려가는 부분
  3. ascender : 기준선에서 올라가는 부분

블록 요소 안에 이미지가 있을 때 이미지 하단 공백처리 (descender 부분)

> inline 요소는 descender 때문에 보이지 않는 행을 생성



- descender 없애는 방법
  1. 인라인 요소를 블록 처리
     - img{display:block}
  2. 이미지의 기준선을 박스안의 상단이나 하단 중앙으로 맞춤 
     - vertical-align: top / middle / bottom 인라인 요소에 직접주는 기준, 인라인 요소는 기본적으로 베이스라인이 기준)

<style>
	.img-title img{width:100%; vertical-align: top;}

## 테이블 태그

**HTML `<table>` 요소**는 행과 열로 이루어진 표를 나타냅니다.

> 사용범위: 네이버 주식 페이지, 네이버 날씨 데이터
>
> 음성으로 읽어주는 등 접근성을 고려하여 표는 쉽지만 지켜야 될 것이 많다

![table_Ex1](D:\SeSSac\03\ex\table_Ex1.jpg)



- table 구성

```
<table> 표
<caption> 표의 설명 또는 제목
<colgroup> 표의 열을 묶는 그룹을 정의 (<col>, <row>)
<thead> 테이블 헤드(생략가능)
<tbody> 헤드가 있으면 바디가 반드시 있음
<tr> 행 줄
<td> 칸 셀
<th> 제목 칸 셀
th속성: scope="row(행의 제목) / col(열의 제목)"
<tfoot> 맺음말(생략가능)
```

border-collapse

```html
.bbs3{width: 700px; height: 400px; border-collapse: collapse;}
```

> 테이블 속성은 디폴트로 border-collapse가 seperate로 되어 있음



- 순서 선택자

  1. 선택자:first-child
     부모를 기준으로 첫번 째 자식

  2. 선택자: last-child
     부모 기준으로 마지막 자식

     > first나 last를 쓰는 이유는 목록 추가등의 확장성 때문

  3. 선택자: nth-child(n)
     부모 기준으로 n번째 자식

     > n+1, 2n+1 등으로 사용 가능 
     >
     > 인덱스 n은 0부터

  4. 선택자: nth-of-type(n)
     같은 타입 중에 n번째

```html
<ul>
    <li></li> <!-- li: first-child -->
    <li></li> <!-- li:nth-child(2) -->
    <li></li>
    <li></li>
    <li></li>
    <li></li> <!-- li:last-child -->
</ul>
<dl>
    <dt></dt> <!-- dt:nth-child(1) / dt:first-child -->
    <dd></dd> <!-- dd:nth-child(2) / dd:nth-of-type(1)-->
    <dt></dt> <!-- dt:nth-child(2) -->
    <dd></dd>
    <dt></dt>
</dl>
```

## float

CSS 속성(property) **`float`** 은 한 요소(element)가 보통 흐름(normal flow)으로부터 빠져 텍스트 및 인라인(inline) 요소가 그 주위를 감싸는 자기 컨테이너의 좌우측을 따라 배치되어야 함을 지정합니다.

> 블록은 한 행이 자신의 컨텐츠기 때문에 나열 시 옆으로 오지 않음



- float:
  1. left
  2. right

> 현재 위치에 띄워서(float) 왼쪽에  붙이거나 오른쪽으로 붙임



### clear

**`clear`** [CSS](https://developer.mozilla.org/ko/docs/Web/CSS) 속성은 요소가 선행 [부동](https://developer.mozilla.org/ko/docs/Web/CSS/float)(floating) 요소 다음일 수 있는지 또는 그 아래로 내려가(해제되어(cleared))야 하는 지를 지정합니다. `clear` 속성은 부동 및 비부동 요소 모두에 적용됩니다.

비부동 블록에 적용되는 경우, 모든 관련 부동체(floats)의 [마진 경계](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_Box_Model/Introduction_to_the_CSS_box_model) 아래로까지 요소의 [보더 경계](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_Box_Model/Introduction_to_the_CSS_box_model)를 아래로 내립니다. 이 움직임(이 일어나는 경우)은 [마진 상쇄](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_Box_Model/Mastering_margin_collapsing)가 일어나지 않도록 합니다.

부동 요소에 적용되는 경우, 모든 관련 부동체의 [마진 경계](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_Box_Model/Introduction_to_the_CSS_box_model) 아래로 요소의 [마진 경계](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_Box_Model/Introduction_to_the_CSS_box_model)를 옮깁니다. 이는 나중(later) 부동체의 위치에 영향을 줍니다, 나중 부동체가 앞선(earlier) 부동체보다 높게 위치될 수 없기에.



- clear:
  1. none
  2. left
  3. right
  4. both



- clear: both

> float 사용 이후의 컨텐츠를 안끌려가게 함	

하위 요소를 float 설정하면, 상위 요소의 heigt값이 지정되어있지 않을 시 both:0이 됨



- float으로 끌려오는게 싫을 경우

1. 상위 요소의  height 값을 설정
2. overflow: hidden 속성 이용
3. float을 float으로 대응

> 상위 하위 모두 float으로 사용할 경우 때에 따라 안 잡히거나 문제가 생길 수도 있음

4. clearfix를 사용

> 마지막 요소를 float을 쓰지 않은 것을 클리어픽스라고 함

> 3, 4번 권장
